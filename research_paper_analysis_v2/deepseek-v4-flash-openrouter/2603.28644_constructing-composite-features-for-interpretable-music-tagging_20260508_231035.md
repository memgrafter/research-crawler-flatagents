# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts
- arXiv ID: 2603.28644
- Source URL: https://arxiv.org/abs/2603.28644
- Reference count: 0
- Primary result: GP-augmented features achieve up to 0.730 ROC-AUC on MTG-Jamendo and 0.805 accuracy on GTZAN, surpassing baselines of 0.727 and 0.765 respectively.

## Executive Summary
This paper presents a Genetic Programming (GP) pipeline that automatically constructs interpretable composite features for music tagging by mathematically combining base audio features. The method addresses the lack of interpretability in deep feature fusion while achieving representational benefits. Most performance gains occur within the first few hundred GP evaluations, indicating modest computational budgets suffice. Top evolved expressions include linear, nonlinear, and conditional forms with low complexity, revealing interpretable feature interactions such as dynamics-tempo and harmonic-genre synergies.

## Method Summary
The pipeline extracts base audio features from audio using Essentia (E23 set of 23 signal-level descriptors) and an extended ALL62 set (E23 plus 32 harmonic-function features via Omnizart/SPARQL and 7 perceptual features via VGG-style CNN). Features are scaled to zero mean and unit variance. GP evolution (using DEAP library) evolves composite features as expression trees combining base features using arithmetic, trigonometric, hyperbolic, min/max, activation functions, and ternary if operators. Population sizes 100 or 500 run for up to 50 generations with early stopping. Fitness is evaluated by training XGBoost on the augmented feature set on a held-out validation split, with parsimony pressure (λ=0.01 per node) applied to control bloat. After evolution, the top composite features augment the base set for final XGBoost training and deployment.

## Key Results
- On MTG-Jamendo (mood/theme tagging, 56 tags), GP-augmented features achieved 0.730 ROC-AUC versus 0.727 baseline.
- On GTZAN (genre tagging, 10 genres), accuracy reached 0.805 versus 0.765 baseline.
- Most performance gains occurred within the first few hundred GP evaluations, with diminishing returns thereafter.

## Why This Works (Mechanism)

### Mechanism 1: Rapid early gains under modest compute budgets
GP evolves composite features by mathematically combining base features. Early generations explore diverse combinations, yielding rapid gains as the search discovers beneficial feature interactions. Later refinement produces diminishing returns as the search approaches local optima. The core assumption is that evolutionary search can discover sparse feature interactions that improve predictive utility within limited evaluations.

Evidence anchors: The abstract states "most of the performance gains are noticed within the first few hundred GP evaluations." Section 3.3 confirms "the best-so-far curves rise steeply during the first few hundred evaluations and then flatten."

Break condition: If the base feature set already captures all relevant interactions, or the GP search space is too restricted, improvements will be negligible.

### Mechanism 2: Interpretability through symbolic expressions
Unlike deep models that learn opaque feature hierarchies, GP produces explicit symbolic expressions showing how base features combine (e.g., `Loudness - BPM`). Each expression is a human-readable tree of operators and terminals, enabling direct analysis of feature interactions. Parsimony pressure favors simpler expressions, maintaining interpretability.

Evidence anchors: The abstract mentions "capturing synergistic interactions while preserving interpretability." Section 4.1 reports "GP discovers heterogeneous solutions... with various low-complexity solutions at top performance."

Break condition: If domain experts cannot interpret evolved expressions (due to unintuitive operator combinations), or if parsimony pressure discards genuinely useful complex expressions.

### Mechanism 3: Co-occurrence analysis reveals music-theoretical synergies
Analyzing pairwise feature co-occurrence and operator-feature patterns across top-evolved expressions identifies which base features and transformations tend to co-appear with high predictive performance for specific tagging tasks. Statistical regularities in high-performing GP expressions reflect real musical relationships.

Evidence anchors: The abstract states "Analyzing these composite features further reveals which interactions and transformations tend to be beneficial for tagging." Section 4.2 reports "GTZAN exhibits... harmonic function pairs... reflecting genre-distinctive chord progression patterns."

Break condition: If co-occurrence patterns are dominated by normalization artifacts or random search history, analysis may produce misleading musicological conclusions.

## Foundational Learning

### Concept 1: Genetic Programming (GP)
Evolutionary algorithm where individuals are expression trees composed of operators and terminals, evolved via crossover and mutation.

- Why needed here: The entire pipeline relies on GP to generate interpretable composite features.
- Quick check question: What is the difference between a GP expression tree node that is an operator (e.g., `cos`) versus a terminal (e.g., `BPM`)?
  - Answer: Operator nodes take child subtrees as inputs and perform a function; terminal nodes are leaf nodes representing base features or constants.

### Concept 2: Parsimony pressure
A penalty term added to fitness that favors simpler (shorter) expressions.

- Why needed here: The paper explicitly uses λ=0.01 per node to control bloat and promote interpretability.
- Quick check question: If two features have the same validation performance but different expression sizes, which does the GP prefer, and why?
  - Answer: The smaller expression, because parsimony pressure penalizes larger trees, favoring simpler solutions.

### Concept 3: XGBoost with multilabel/multiclass classification
A gradient-boosted tree ensemble used as the downstream classifier.

- Why needed here: The GP pipeline evaluates composite features by training XGBoost on augmented feature sets; understanding its hyperparameters is critical for reproducibility.
- Quick check question: How might the XGBoost max depth interact with composite features that are already nonlinear combinations?
  - Answer: Shallow trees (max depth 2-3) may not fully capture interactions already encoded in composite features, while deeper trees could redundantly model the same patterns.

## Architecture Onboarding

### Component map
Input audio -> Essentia feature extractor (E23/ALL62) -> feature scaling -> GP evolution (DEAP library) -> composite feature expressions -> XGBoost classifier -> tag predictions.

Key component boundaries: feature extraction is offline; GP runs on extracted features; XGBoost sees both base and composite features.

### Critical path
(1) Scaling must be consistent between GP-fitness evaluation and final deployment.
(2) The GP fitness function must mirror the final evaluation metric (ROC-AUC or accuracy) exactly.
(3) Termination criteria (stagnation/convergence) determine search length.

### Design tradeoffs
Population size (100 vs 500) affects search diversity vs. compute time. Parsimony penalty strength (λ=0.01) balances interpretability vs. ability to represent complex interactions. Using XGBoost as classifier prioritizes robustness over potential performance from deeper fusion.

### Failure signatures
(1) NaN or infinite values in composite features due to unprotected operations (protected primitives should prevent this).
(2) No improvement after first few hundred evaluations indicates either the search space is exhausted or base features already capture all signal.
(3) Expressions with very high complexity but poor performance suggest bloat not controlled by parsimony penalty.

## Open Questions the Paper Calls Out

### Open Question 1
Do the GP-evolved composite features generalize to other music tagging tasks beyond mood/theme and genre (e.g., instrument or era tagging)? The paper only evaluates on MTG-Jamendo and GTZAN, leaving the generalization question unresolved. Evidence to resolve: Apply the pipeline to Medley-solos-DB for instrument tagging or a decade-tagging dataset.

### Open Question 2
How sensitive are the GP-augmented performance gains to the choice of base classifier (XGBoost) and its hyperparameters? The paper uses XGBoost without exploring alternatives like random forests or SVMs. Evidence to resolve: Replace XGBoost with logistic regression, gradient boosting, or SVM and compare GP-augmented vs. baseline performance across multiple hyperparameter configurations.

### Open Question 3
Can the GP pipeline produce composite features that remain interpretable and effective when scaled to larger base feature sets (e.g., >200 dimensions)? The paper only tests with E23 and ALL62 (≤62 dimensions), noting the method provides representational benefits within this limited scope. Larger feature spaces would increase combinatorial complexity. Evidence to resolve: Extend the pipeline to 500+ dimensional feature sets from Essentia's full extractor and measure performance and expression complexity.

## Limitations
- Reproducibility of exact held-out validation splits and the number of composite features evolved per iteration (M) are unspecified, limiting precise replication.
- Reported improvements on MTG-Jamendo (0.003 AUC delta) are small; statistical significance is not clearly established.
- Interpretability claim rests on human-readability of expressions, but no user study validates whether domain experts find them interpretable or actionable.

## Confidence
- **High:** GP improves accuracy over baseline on GTZAN (0.765→0.805, clear delta). Mechanism 1 (rapid early gains) is well-supported by learning curves and moderate evaluation budgets.
- **Medium:** GP improves AUC on MTG-Jamendo (very small delta, may be unstable). Interpretability claim (Mechanism 2) is plausible but unvalidated externally.
- **Low:** Co-occurrence analysis revealing music-theoretical synergies (Mechanism 3) — conclusions may be overinterpreted given small expression samples and unknown noise.
