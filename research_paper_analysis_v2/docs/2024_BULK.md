# 2024_BULK.md

## Goal
Bulk-download **2024 arXiv PDFs** to mounted HDD and use cache-first local reads.

## Canonical arXiv bulk path
- Bucket: `s3://arxiv` (**Requester Pays**)
- Region: `us-east-1`
- PDF manifest: `s3://arxiv/pdf/arXiv_pdf_manifest.xml`
- Chunks: `pdf/arXiv_pdf_YYMM_NNN.tar` (~500MB each)

> Important: S3 objects are chunked by month/sequence, **not by category**.
> You cannot directly do `s3://arxiv/pdf/cs.CL/...`.

## 1) Prereqs
```bash
aws configure sso   # or aws configure
aws sts get-caller-identity
```

## 2) Download manifest + compute exact 2024 totals
```bash
mkdir -p /mnt/linux_hdd/arxiv_bulk_2024/meta
cd /mnt/linux_hdd/arxiv_bulk_2024/meta

aws s3 cp s3://arxiv/pdf/arXiv_pdf_manifest.xml ./arXiv_pdf_manifest.xml \
  --request-payer requester --region us-east-1

python - <<'PY'
import xml.etree.ElementTree as ET
from pathlib import Path

root = ET.parse('arXiv_pdf_manifest.xml').getroot()
keys, total = [], 0
for f in root.findall('.//file'):
    yymm = (f.findtext('yymm') or '').strip()
    fn = (f.findtext('filename') or '').strip()
    sz = int((f.findtext('size') or '0').strip())
    if yymm.startswith('24') and fn.startswith('pdf/arXiv_pdf_'):
        keys.append(fn)
        total += sz

Path('2024_pdf_keys.txt').write_text('\n'.join(keys) + '\n')
Path('2024_total_bytes.txt').write_text(str(total) + '\n')
print('files:', len(keys))
print('bytes:', total)
print('GB:', round(total/1e9, 2), 'TB:', round(total/1e12, 3))
PY
```

## 3) Full 2024 download (all chunks)

## Selective mode: ~14k papers from 5–9 categories
This is the size-tuning workflow.

### What this means for cost (important)
Because S3 is chunked into month tar files, category-selected IDs are often spread across many chunks.
So selective mode can still require a large fraction of 2024 tar files.

- **Best case (IDs concentrated in fewer chunks):** much smaller than full-year.
- **Common case (IDs spread across months/categories):** close to full 2024 chunk volume.

Use `selected_2024_total_bytes.txt` from the mapping step for exact cost before downloading.

Quick egress estimate at `$0.09/GB`:
- 100 GB -> ~$9
- 300 GB -> ~$27
- 600 GB -> ~$54
- 1.0 TB -> ~$90
- 1.4 TB -> ~$126

### A) Build target arXiv ID list from local DB (example)
(Adjust categories and ordering.)
```bash
sqlite3 arxiv_crawler/data/arxiv.sqlite "
SELECT DISTINCT arxiv_id
FROM papers
WHERE arxiv_id LIKE '24%'
  AND primary_category IN ('cs.CL','cs.CV','cs.AI','cs.IR','stat.ML')
ORDER BY published_at DESC
LIMIT 14000;
" > /mnt/linux_hdd/arxiv_bulk_2024/meta/selected_2024_ids.txt
```

### B) Map IDs -> minimal required S3 tar chunks
```bash
cd /mnt/linux_hdd/arxiv_bulk_2024/meta
python - <<'PY'
import re
import xml.etree.ElementTree as ET
from pathlib import Path

ids = [x.strip().split('v')[0] for x in Path('selected_2024_ids.txt').read_text().splitlines() if x.strip()]
root = ET.parse('arXiv_pdf_manifest.xml').getroot()

# Build month->[(start,end,filename,size)] from manifest.
by_month = {}
for f in root.findall('.//file'):
    yymm = (f.findtext('yymm') or '').strip()
    fn = (f.findtext('filename') or '').strip()
    sz = int((f.findtext('size') or '0').strip())
    first = (f.findtext('first_item') or '').strip().split('v')[0]
    last  = (f.findtext('last_item') or '').strip().split('v')[0]

    m1 = re.match(r'^(\d{4})\.(\d+)$', first)
    m2 = re.match(r'^(\d{4})\.(\d+)$', last)
    if not (yymm.startswith('24') and fn.startswith('pdf/arXiv_pdf_') and m1 and m2):
        continue
    if m1.group(1) != yymm or m2.group(1) != yymm:
        continue
    by_month.setdefault(yymm, []).append((int(m1.group(2)), int(m2.group(2)), fn, sz))

for k in by_month:
    by_month[k].sort()

need = set()
missing = []
for aid in ids:
    m = re.match(r'^(\d{4})\.(\d+)$', aid)
    if not m:
        continue
    yymm, n = m.group(1), int(m.group(2))
    found = False
    for lo, hi, fn, sz in by_month.get(yymm, []):
        if lo <= n <= hi:
            need.add((fn, sz))
            found = True
            break
    if not found:
        missing.append(aid)

files = sorted(need)
total = sum(sz for _, sz in files)
Path('selected_2024_tar_keys.txt').write_text('\n'.join(fn for fn, _ in files) + '\n')
Path('selected_2024_total_bytes.txt').write_text(str(total) + '\n')

print('target_ids:', len(ids))
print('needed_tar_files:', len(files))
print('needed_bytes:', total)
print('needed_GB:', round(total/1e9, 2), 'needed_TB:', round(total/1e12, 3))
print('unmapped_ids:', len(missing))
PY
```

## Cost + timeline quick calculator (exact bytes)
```bash
cd /mnt/linux_hdd/arxiv_bulk_2024/meta
python - <<'PY'
from pathlib import Path

# switch file name depending on full vs selective
total = int(Path('selected_2024_total_bytes.txt').read_text().strip())  # or 2024_total_bytes.txt
USD_PER_GB = 0.09      # adjust to current us-east-1 pricing tier
EFFECTIVE_MBPS = 500   # sustained throughput

gb = total / 1e9
usd = gb * USD_PER_GB
seconds = total * 8 / (EFFECTIVE_MBPS * 1_000_000)

print(f'total_gb={gb:.2f}')
print(f'est_egress_usd=${usd:.2f}')
print(f'est_time_hours={seconds/3600:.2f} @ {EFFECTIVE_MBPS} Mbps effective')
PY
```
