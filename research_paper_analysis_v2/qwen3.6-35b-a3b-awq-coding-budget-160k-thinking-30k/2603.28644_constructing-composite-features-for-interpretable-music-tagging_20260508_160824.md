# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts
- arXiv ID: 2603.28644
- Source URL: https://arxiv.org/abs/2603.28644
- Reference count: 0
- Primary result: GP-evolved composite features improve ROC-AUC and classification accuracy on MTG-Jamendo and GTZAN while preserving full symbolic interpretability.

## Executive Summary
Deep learning models for music tagging often sacrifice interpretability for performance. This work addresses that gap by introducing a Genetic Programming (GP) pipeline that automatically evolves mathematical composite features from standard audio descriptors. By optimizing candidate feature combinations against XGBoost validation performance and applying parsimony pressure, the method enforces structural simplicity without sacrificing predictive power.

Evaluations on the MTG-Jamendo and GTZAN datasets demonstrate consistent improvements over baseline interpretable models. The approach converges rapidly within a few hundred GP generations, producing optimal expressions that span linear, nonlinear, and conditional forms. The results indicate that explicit symbolic feature construction can match deep fusion accuracy while retaining transparent, human-readable decision logic.

## Method Summary
The pipeline begins with a set of standard audio descriptors, which are treated as terminal nodes for symbolic regression. Genetic Programming iteratively evolves mathematical expression trees that combine these descriptors. Each candidate solution is evaluated by training an XGBoost regressor/classifier on the derived features, with validation performance serving as the fitness metric.

To prevent overfitting and maintain interpretability, a parsimony penalty is applied during selection, favoring shorter and structurally simpler expressions. The process halts when performance plateaus or a generation limit is reached, yielding a final set of evolved composite features that directly substitute for handcrafted descriptors in downstream tagging tasks.

## Key Results
- MTG-Jamendo ROC-AUC improved from 0.727 to 0.730 (ALL62 base) and 0.719 to 0.724 (E23 base).
- GTZAN classification accuracy increased from 0.765 to 0.805 and 0.740 to 0.790 across respective baselines.
- Optimal feature expressions converged within 300–1,000 GP evaluations, consistently outperforming standard interpretable baselines.

## Why This Works (Mechanism)
The pipeline works by decoupling feature engineering from model training. Instead of relying on fixed audio descriptors, GP explores a vast symbolic space to discover non-linear interactions and conditional dependencies that better capture musical structure. The XGBoost fitness proxy ensures that evolved features directly contribute to discriminative power, while parsimony pressure prevents expression bloat. This creates a transparent mapping where each mathematical operation corresponds to a specific audio-to-tag relationship, preserving causal traceability that black-box deep models obscure.

## Foundational Learning
- Symbolic Regression / Genetic Programming: Why needed: Explores mathematical combinations of base features to discover interpretable non-linear relationships. Quick check: Monitor tree depth and parsimony penalty weight to avoid overfitting.
- XGBoost Fitness Proxy: Why needed: Provides a fast, robust evaluation metric during evolution without requiring full end-to-end training. Quick check: Validate that cross-validated XGBoost performance correlates with final tagging task performance.
- Parsimony Pressure: Why needed: Enforces structural simplicity, ensuring evolved expressions remain human-readable and computationally efficient. Quick check: Track average expression length across generations to verify complexity suppression.
- Audio Descriptor Initialization: Why needed: Grounds the evolutionary search in domain-specific, physically meaningful features. Quick check: Verify descriptor coverage across frequency, temporal, and harmonic domains.

## Architecture Onboarding
- Component Map: Audio Descriptor Extraction -> GP Expression Tree Evolution -> XGBoost Fitness Evaluation -> Parsimony Selection -> Final Composite Feature Output
- Critical Path: Descriptor extraction feeds directly into GP initialization; evaluation loop (tree generation -> XGBoost validation -> fitness ranking) dictates evolutionary trajectory; parsimony pruning ensures final feature set remains compact.
- Design Tradeoffs: Higher parsimony weights improve interpretability but may restrict expressive capacity; lower weights capture complex interactions at the cost of transparency and longer convergence times.
- Failure Signatures: Expression bloat beyond parsimony thresholds; XGBoost validation divergence from test metrics indicating overfitting to evolution proxy; convergence stagnation due to inadequate descriptor diversity.
- First Experiments: 1) Run GP with varying parsimony weights (1.0, 1e-3, 1e-5) to map interpretability vs accuracy. 2) Test descriptor subsets (spectral-only vs harmonic-only) to isolate feature contribution. 3) Compare XGBoost fitness proxy correlation against end-to-end deep model validation scores.

## Limitations
- Evaluation is restricted to MTG-Jamendo and GTZAN, limiting claims about cross-domain generalization.
- Direct comparisons against modern deep fusion architectures are absent, making the matching deep accuracy assertion preliminary.
- Symbolic transparency may degrade as tree depth increases under relaxed parsimony constraints, obscuring causal feature relationships.

## Confidence
- High: Metric improvements over standard interpretable baselines are robustly demonstrated across both datasets.
- Medium: Cross-dataset generalizability and equivalence to deep learning systems remain preliminary without broader benchmarking.
- Low: Long-term robustness and real-world deployment stability lack external validation.

## Next Checks
