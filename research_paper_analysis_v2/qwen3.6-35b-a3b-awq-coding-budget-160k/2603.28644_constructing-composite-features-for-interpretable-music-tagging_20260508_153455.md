# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts
- **arXiv ID**: 2603.28644
- **Source URL**: https://arxiv.org/abs/2603.28644
- **Reference Count**: 0
- **Primary Result**: Genetic Programming-evolved composite features achieved 0.730 AUC on MTG-Jamendo and 80.5% accuracy on GTZAN.

## Executive Summary
Music tagging systems often rely on deep learning feature fusion, which frequently sacrifices interpretability for performance. This study addresses that limitation by proposing a Genetic Programming (GP) pipeline to automatically evolve composite features from base audio descriptors. The approach balances predictive power with human-readable symbolic expressions, allowing for explicit insight into how audio characteristics interact.

Experiments on the MTG-Jamendo and GTZAN datasets demonstrate that GP-augmented features consistently outperform state-of-the-art interpretable baselines. The method achieves performance gains rapidly within the first few hundred evaluations, indicating high computational efficiency while providing transparent, symbolic representations of synergistic audio traits that are typically opaque in black-box models.

## Method Summary
The proposed method utilizes a Genetic Programming pipeline to evolve composite features from a set of base audio descriptors. Instead of relying on fixed feature engineering or opaque neural network layers, the system automatically generates symbolic expressions that combine these base descriptors. These expressions are evolved to optimize tagging performance while maintaining a structure that is human-readable.

The pipeline evaluates candidate expressions based on their ability to predict music tags, selecting and mutating combinations that yield the best performance. This process results in a library of composite features that capture complex linear, nonlinear, and conditional interactions between raw audio features, effectively bridging the gap between high-performance feature fusion and interpretability.

## Key Results
- Achieved 0.730 AUC on the MTG-Jamendo dataset and 80.5% accuracy on the GTZAN dataset, surpassing state-of-the-art interpretable baselines.
- Performance improvements emerged rapidly within the first few hundred GP evaluations, demonstrating high computational efficiency relative to the gains.
- Evolved expressions revealed clear linear, nonlinear, and conditional interactions, providing explicit symbolic insights into feature synergy.

## Why This Works (Mechanism)
The effectiveness of this approach stems from the explicit construction of feature interactions through symbolic evolution. By using Genetic Programming, the model avoids the "black box" nature of deep learning fusion, instead generating expressions that can be analytically inspected. This allows for the discovery of specific, non-obvious relationships between base audio descriptors that are directly tied to the tagging task. The mechanism relies on the search space of symbolic mathematics to find compact, efficient representations of complex acoustic patterns, ensuring that every component of the final feature set has a defined logical role.

## Architecture Onboarding
- **Component Map**: Base Audio Descriptors -> GP Evolution Engine -> Evolved Composite Expressions -> Music Tagging Model -> Predictions.
- **Critical Path**: The bottleneck lies in the GP evaluation phase, where each candidate expression must be tested against the tagging model. Efficiency gains are driven by rapid convergence in early generations.
- **Design Tradeoffs**: The system trades the absolute peak performance potential of deep neural networks for explicit interpretability and faster feature development times compared to manual engineering.
- **Failure Signatures**: Overfitting to dataset-specific noise may result in highly complex expressions that do not generalize; lack of diversity in the initial population may limit the range of discovered interactions.
- **First Experiments**:
  1. Run GP evolution on a small subset of MTG-Jamendo to monitor convergence behavior and expression complexity.
  2. Compare the accuracy of single evolved features versus aggregated feature sets on GTZAN.
  3. Perform a manual inspection of top-performing evolved expressions to validate their logical coherence against acoustic principles.
