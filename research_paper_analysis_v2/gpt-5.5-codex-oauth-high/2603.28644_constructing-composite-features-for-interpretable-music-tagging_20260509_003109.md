# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts

- **arXiv ID:** 2603.28644
- **Source URL:** https://arxiv.org/abs/2603.28644
- **Reference count:** 0
- **Primary result:** Genetic Programming (GP) evolves explicit composite audio features that, when added to XGBoost music-tagging models, consistently improve MTG-Jamendo AUC and GTZAN accuracy while keeping feature interactions inspectable.

## Executive Summary

This paper targets the interpretability gap in deep feature-fusion methods for music tagging. Instead of learning opaque embeddings, it uses Genetic Programming to construct symbolic mathematical features from existing audio descriptors, then adds those features to XGBoost classifiers.

Across MTG-Jamendo mood/theme tagging and GTZAN genre classification, GP-augmented models improve over baselines for both a 23-feature Essentia set and a 62-feature perceptual/harmonic set. The gains are modest for MTG-Jamendo but larger for GTZAN, and the resulting expressions can be inspected as linear, nonlinear, and conditional feature interactions.

## Method Summary

The pipeline starts from standardized base feature matrices: **E23**, a 23-feature Essentia signal-level descriptor set, and **ALL62**, which adds harmonic-function and perceptual CNN-regressed features. The method evaluates two tasks: MTG-Jamendo mood/theme multilabel tagging using ROC-AUC, and GTZAN 10-class genre classification using accuracy.

GP evolves scalar expression trees whose terminals are base features and constants, and whose internal nodes are mathematical or conditional operators. Each candidate composite feature is appended to the current feature set, standardized, and scored by validation-set XGBoost performance minus a parsimony penalty of λ = 0.01 per tree node. Up to five composite features are added iteratively, after which the selected augmented feature set is evaluated on the held-out test set.

## Key Results

- **ALL62 features:** MTG-Jamendo AUC improves from **0.727 → 0.730**; GTZAN accuracy improves from **0.765 → 0.805**.
- **E23 features:** MTG-Jamendo AUC improves from **0.719 → 0.724**; GTZAN accuracy improves from **0.740 → 0.790**.
- **Search efficiency:** most gains appear early, with near-optimal GTZAN performance around **300 evaluations** and substantial MTG-Jamendo gains around **1,000 evaluations**.

## Why This Works (Mechanism)

### 1. GP exposes useful feature interactions

- **Claim:** GP augmentation can improve tagging when predictive interactions among existing descriptors are not already easy for the classifier to exploit.
- **Mechanism:** GP builds explicit scalar expressions from base features using arithmetic, nonlinear, and conditional operations. If a composite feature captures a discriminative interaction, XGBoost can split on that single constructed dimension more directly than on multiple raw descriptors.
- **Core assumption:** The base descriptors contain complementary information whose interactions are predictive but under-exposed to shallow tree ensembles.
- **Evidence:** The paper reports consistent gains across MTG-Jamendo and GTZAN; Table-level results show sizable GTZAN improvements and smaller MTG-Jamendo improvements.
- **Break condition:** The mechanism may fail if the base descriptors are already saturated, if labels depend on information absent from the features, or if validation performance rewards dataset-specific artifacts.

### 2. Symbolic expressions preserve inspectability

- **Claim:** The method offers more interpretability than deep feature-fusion approaches because learned transformations remain symbolic.
- **Mechanism:** Each GP individual is an expression tree with named input features and explicit operators. Practitioners can inspect whether a feature depends on loudness, BPM, harmony, spectral entropy, mode, chord changes, or conditional logic.
- **Core assumption:** Human-readable symbolic formulas are useful enough for MIR practitioners to reason about feature interactions.
- **Evidence:** The paper analyzes evolved expressions including linear, nonlinear, and conditional forms, as well as feature-feature and feature-operator co-occurrence patterns.
- **Break condition:** Interpretability weakens if expressions become large, numerically unstable, dominated by obscure descriptors, or over-interpreted as causal music-theoretic explanations.

### 3. Early evolutionary search can find simple high-value features

- **Claim:** Much of the benefit can appear under modest GP budgets.
- **Mechanism:** Early generations sample diverse expression trees, score them directly against validation performance, and retain useful structures through tournament selection, crossover, and mutation. If the search space contains many low-complexity improvements, useful formulas can emerge before long runs.
- **Core assumption:** Helpful interactions are not rare, high-depth formulas requiring extensive search.
- **Evidence:** The paper reports steep-then-plateau trajectories, with near-optimal GTZAN performance within about 300 evaluations and substantial MTG-Jamendo gains around 1,000 evaluations.
- **Break condition:** Early gains may disappear with noisy validation splits, insufficient population diversity, mismatched operators, or tasks requiring high-order interactions.

## Foundational Learning

### Genetic Programming expression trees

- **Why needed:** The core learned artifact is a symbolic tree, not a neural embedding. Terminals are audio features or constants; internal nodes are operators such as arithmetic functions, `log`, `sqrt`, `max`, `sigmoid`, or conditionals.
- **Quick check:** Can you draw one evolved expression as a tree and identify its terminals, operators, depth, and node count?

### Music audio descriptors and perceptual features

- **Why needed:** The evolved formulas are only meaningful if the base descriptors are understood, including loudness, BPM, onset rate, spectral flux, spectral entropy, harmonic-function n-grams, and perceptual properties such as danceability.
- **Quick check:** Which E23 features are signal-level descriptors, and which ALL62 additions represent harmonic or perceptual information?

### Validation-driven feature construction with XGBoost

- **Why needed:** GP fitness is computed by adding a candidate feature and measuring held-out XGBoost performance, so split discipline, metric choice, and leakage prevention are central.
- **Quick check:** Why must GP fitness use only training/validation data and never the final test set?

### Parsimony and expression complexity

- **Why needed:** The method penalizes larger expression trees to balance performance against readability and overfitting risk.
- **Quick check:** What happens if the node-count penalty is too weak or too strong?

## Architecture Onboarding

### Component map

Audio files -> E23 Essentia feature extraction and ALL62 perceptual/harmony feature construction -> feature standardization -> GP expression-tree candidate generation -> candidate sanitization and standardization -> XGBoost validation scoring -> parsimony-adjusted fitness -> iterative selection of up to 5 composite features -> final XGBoost model training -> held-out test evaluation and expression analysis

### Critical path

Reproduce the baseline XGBoost scores first, then freeze train/validation/test splits. Implement candidate expression evaluation safely, including protected operators and invalid-value handling. Add one composite feature per GP iteration, score it only on validation data, and retrain final deployment models only after feature selection is complete.

### Design tradeoffs

Larger GP populations improve exploration but raise cost. Richer operator sets may discover stronger transformations but can reduce interpretability and increase numerical instability. Stronger parsimony improves readability but may suppress useful complex interactions. Direct validation optimization is practical but can overfit if the search is too aggressive.

### Failure signatures

Validation performance improves while test performance drops. Many candidates produce NaN or infinite values before sanitization. Selected expressions collapse to constants, duplicates, or near-copies of base features. Tree depth grows without metric gains. Confidence intervals overlap heavily while small improvements are over-emphasized. GP repeatedly selects dataset artifacts such as track length.

## Open Questions the Paper Calls Out

### 1. Do GP-constructed composite features generalize beyond these datasets?

- **Basis:** Experiments use MTG-Jamendo mood/theme tagging and GTZAN genre tagging.
- **Why unresolved:** Learned interactions may be dataset-, culture-, genre-, or tag-vocabulary-specific.
- **Evidence needed:** Cross-dataset evaluation on additional MIR benchmarks with consistent gains and reusable expressions.

### 2. Are the small MTG-Jamendo gains statistically and practically meaningful?

- **Basis:** Reported MTG-Jamendo improvements are about 0.002–0.005 AUC.
- **Why unresolved:** Bootstrap intervals are shown, but significance of GP versus baseline is not directly established.
- **Evidence needed:** Paired significance tests and per-tag analyses showing reliable improvement beyond variance.

### 3. How sensitive is performance to GP design choices?

- **Basis:** Many GP hyperparameters are fixed, including primitive set, population size, parsimony penalty, and number of evolved features.
- **Why unresolved:** No ablation isolates which settings drive the gains.
- **Evidence needed:** Controlled ablations reporting performance, expression complexity, search cost, and stability.

### 4. Do human music experts find the evolved expressions genuinely interpretable?

- **Basis:** The paper analyzes symbolic expressions and discusses music-theoretic implications.
- **Why unresolved:** Interpretability is argued from symbolic form, not measured with human subjects.
- **Evidence needed:** Expert studies rating understandability, plausibility, and usefulness of GP features versus baseline descriptors or learned embeddings.

## Limitations

- Exact splits, seeds, preprocessing exclusions, and full ALL62 construction details are not fully specified, limiting faithful reproduction.
- MTG-Jamendo gains are small, so statistical significance and sensitivity to split choice remain uncertain.
- Because GP optimizes validation performance, overfitting to validation artifacts is a real risk.

## Confidence

- **Reported in-paper performance gains:** High
- **Generalization of gains beyond the tested datasets:** Low
- **Interpretability advantage of symbolic features:** Medium
- **Early-search efficiency:** Medium
- **Faithful reproduction without released splits/features:** Low
