# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts
* **arXiv ID:** 2603.28644
* **Source URL:** https://arxiv.org/abs/2603.28644
* **Reference Count:** 0
* **Primary Result:** GP-evolved composite features outperform state-of-the-art interpretable baselines, improving GTZAN accuracy by 4.0%–5.0%.

## Executive Summary
Current deep learning models for music tagging suffer from a "black box" problem, where complex feature fusion lacks transparency. This research proposes a Genetic Programming (GP) pipeline designed to bridge the gap between high performance and human readability.

By using symbolic regression to mathematically combine base audio descriptors, the method evolves composite features that capture synergistic interactions between signal-level and perceptual attributes. The result is a tagging system that provides actionable music-theoretic insights through human-readable mathematical expressions.

## Method Summary
The methodology employs a Genetic Programming (GP) pipeline to automate the discovery of interpretable feature combinations. It starts with a base set of audio descriptors, specifically signal-level E23 features and perceptual ALL62 features.

Through symbolic regression, the GP evolves mathematical expressions—ranging from simple linear combinations to complex nonlinear and conditional forms—that represent these base descriptors. This process optimizes for the ability to distinguish musical tags while ensuring the resulting expressions remain syntactically simple enough for human interpretation.

## Key Results
* **Accuracy Gains:** Achieved a 4.0%–5.0% improvement over interpretable baselines on the GTZAN dataset.
* **Computational Efficiency:** Most performance gains were realized within the first few hundred GP evaluations.
* **Feature Versatility:** Successfully evolved a range of feature types, including linear, nonlinear, and conditional mathematical forms.

## Why This Works (Mechanism)
The approach works by replacing opaque neural feature fusion with symbolic regression. Instead of letting a deep network learn uninterpretable weights, the GP search space is constrained to mathematical operators. This allows the model to capture "synergistic interactions"—where the relationship between two audio descriptors (e.g., spectral centroid and rhythmic energy) provides more information than either descriptor alone—in a format that can be explicitly read and verified.

## Foundational Learning
* **Genetic Programming (GP):** Needed to search the space of mathematical expressions; check by verifying how mutation/crossover affects expression complexity.
* **Symbolic Regression:** Needed to map input descriptors to a functional mathematical form; check by observing if the output is a readable equation.
* **Audio Descriptors (E23 & ALL62):** Needed as the raw building blocks for features; check by ensuring these represent distinct signal and perceptual qualities.
* **Feature Fusion:** Needed to combine multiple data streams; check by comparing single-descriptor performance vs. composite-descriptor performance.

## Architecture Onboarding
**Component Map**
Base Audio Descriptors (E23 + ALL62) -> Genetic Programming Pipeline (Symbolic Regression) -> Evolved Composite Features -> Music Tagging Classifier

**Critical Path**
The core logic flows from the definition of the initial descriptor set through the GP evolutionary loop to the final generation of the mathematical expression used by the classifier.

**Design Tradeoffs**
* **Interpretability vs. Complexity:** Using more complex nonlinear/conditional forms increases accuracy but risks making the expression too dense for human understanding.
* **Search Breadth vs. Speed:** A wider range of mathematical operators improves the chance of finding optimal features but increases the computational cost of the GP iterations.

**Failure Signatures**
* **Overfitting:** Expressions that perform perfectly on training sets but fail on validation sets (often manifested as overly complex/deep equations).
* **Triviality:** The GP settling on simple identity functions or constant values that provide no predictive gain.

**First Experiments**
1. Test the GP on a single-descriptor input to ensure it can recover known mathematical relationships.
2. Measure the correlation between expression depth (complexity) and classification accuracy.
3. Run the pipeline on a subset of features to determine the minimum descriptor set required for meaningful gains.

## Open Questions the Paper Calls Out
None.

## Limitations
* **Generalizability Uncertainty:** It is unclear if evolved expressions remain valid across musical genres not present in the training datasets.
* **Qualitative Interpretability:** The "music-theoretic insights" are qualitative and lack formal proof of alignment with established psychoacoustic axioms.
* **Scalability Unknown:** The computational efficiency is only verified for the initial stages of evolution; scalability for massive, high-dimensional sets is unproven.

## Confidence
* **Performance Gains:** High
* **Interpretability:** Medium
* **Algorithmic Efficiency:** Medium

## Next Checks
1. **Cross-Dataset Robustness:** Evaluate the evolved composite features on a third, structurally distinct dataset (e.g., FMA) to check for overfitting to specific descriptor distributions.
2. **Expert Grounding:** Conduct a blind study with musicologists to determine if the symbolic features correspond to recognized musical attributes (e.g., timbre, rhythm, or texture).
3. **Complexity-Accuracy Trade-off:** Test the model against a "complexity penalty" metric to ensure the GP doesn't favor excessively deep expressions that sacrifice real-world interpretability for marginal gains.
