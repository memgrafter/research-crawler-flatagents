# Constructing Composite Features for Interpretable Music-Tagging

## Quick Facts
- arXiv ID: 2603.28644
- Source URL: https://arxiv.org/abs/2603.28644
- Reference count: 0
- Primary result: A Genetic Programming pipeline automatically evolves interpretable composite features that consistently outperform baseline XGBoost classifiers on standard music-tagging benchmarks.

## Executive Summary
Deep learning models dominate music tagging but often act as black boxes, obscuring how audio features map to perceptual labels. This paper introduces a Genetic Programming (GP) pipeline that automatically evolves composite features by mathematically combining base audio descriptors. By using arithmetic, trigonometric, and conditional operators under parsimony pressure, the method generates compact, human-readable expressions that capture synergistic signal-perceptual relationships.

Augmenting traditional XGBoost classifiers with these evolved features yields measurable accuracy gains on both multilabel mood/theme and multiclass genre datasets. The approach achieves performance competitive with opaque deep fusion methods while fully preserving transparency. Additionally, the evolutionary search converges rapidly, demonstrating that highly effective feature interactions can be discovered with modest computational budgets.

## Method Summary
The pipeline begins by extracting and normalizing base audio descriptors (e.g., Essentia E23 or extended ALL62 sets) from raw audio tracks. These features are fed into a DEAP-based Genetic Programming engine, which initializes a population of expression trees using ramped half-and-half strategies. During evolution, candidate expressions are evaluated by training an XGBoost classifier on the augmented feature set (base features plus the candidate composite).

Fitness is calculated as the XGBoost validation score (ROC-AUC or Accuracy) minus a complexity penalty proportional to the expression tree size. The GP loop applies crossover, mutation, and selection, terminating early if fitness stagnates or variance drops. The top-performing symbolic expressions are extracted, appended to the base feature matrix, and used to train a final XGBoost model with fixed hyperparameters. This separates the heavy training-time search from a lightweight, formula-based inference stage.

## Key Results
- MTG-Jamendo multilabel ROC-AUC improved by 0.002–0.005 over baseline (0.729–0.730 vs. 0.727–0.719).
- GTZAN multiclass accuracy increased by 0.035–0.050 over baseline (0.800–0.805 vs. 0.765–0.740).
- Performance gains converged rapidly, with most improvements captured within the first 300–1000 GP evaluations.

## Why This Works (Mechanism)
### Mechanism 1
- Claim: Genetic Programming may discover non-linear feature interactions that improve tagging performance beyond additive base features, provided the search space includes expressive operators.
- Mechanism: Evolutionary search optimizes expression trees combining base features via arithmetic, trigonometric, and conditional operators. The fitness function evaluates XGBoost performance on augmented feature sets, driving selection toward combinations that capture synergistic signal-perceptual relationships.
- Core assumption: Beneficial feature interactions exist as mathematically expressible transformations within the defined operator set, and XGBoost can leverage these composite inputs effectively.
- Evidence anchors:
  - [abstract] "mathematically combining base music features, thereby capturing synergistic interactions while preserving interpretability"
  - [section] 3.2 reports consistent AUC/ACC improvements across E23 and ALL62 baselines; 4.2 notes co-occurrence patterns like `Spectral-Spread` with timbral features suggest joint contributions to mood.
  - [corpus] Neighbors discuss interpretable features and deep fusion but do not validate GP-based interaction discovery; corpus support is limited to the problem space.
- Break condition: Performance gains may vanish if base features lack latent correlations, if the operator set cannot express the necessary interactions (e.g., missing piecewise logic), or if the target labels are dominated by noise uncorrelated with structural features.

### Mechanism 2
- Claim: Parsimony pressure may yield compact, interpretable expressions without significant accuracy loss, contingent on an appropriate complexity penalty.
- Mechanism: The fitness function includes a term penalizing expression length (node count). This biases the evolutionary search toward simpler expressions that achieve comparable predictive utility, reducing bloat and enhancing human readability.
- Core assumption: Simpler expressions achieving high fitness capture robust signal relationships rather than overfitting to training noise, and that interpretability requirements do not necessitate deep, complex trees.
- Evidence anchors:
  - [abstract] "aligned with parsimony pressure to prefer simpler expressions" and "low-complexity solutions at top performance"
  - [section] 2.2 specifies lambda=0.01 per node; 4.1 observes "Several low-complexity expressions achieve top performance alongside deeper constructs"
  - [corpus] Weak; neighbors highlight interpretability gaps in deep models but do not test parsimony-driven symbolic regression.
- Break condition: If the optimal solution requires high tree depth (e.g., >6 nodes) to model complex dependencies, excessive lambda may force convergence to suboptimal proxies, degrading performance or masking true interactions.

### Mechanism 3
- Claim: The search may converge rapidly within modest computational budgets, suggesting effective combinations are densely distributed in early search phases.
- Mechanism: GP trajectories show a steep-then-plateau pattern, where initial generations explore diverse combinations yielding rapid gains, followed by diminishing returns. This behavior implies that low-hanging fruit interactions are accessible before exhaustive search.
- Core assumption: The feature interaction landscape contains high-performing regions reachable via standard crossover/mutation from random initialization, and dataset complexity bounds the evaluation count needed for saturation.
- Evidence anchors:
  - [abstract] "most of the performance gains are noticed within the first few hundred GP evaluations"
  - [section] 3.3 shows GTZAN near-optimal performance within 300 evaluations and MTG-Jamendo substantial gains within 1000 evaluations; runtime is ~1–5 seconds per evaluation.
  - [corpus] No direct evidence; however, related work on feature engineering suggests modular search spaces often yield early improvements.
- Break condition: Convergence may slow or fail if the dataset is highly ambiguous (e.g., subjective tags with low inter-annotator agreement), if the population size is too small to maintain diversity, or if the search gets trapped in local optima due to rugged fitness landscapes.

## Architecture Onboarding
- Component map:
  Input: Raw audio -> Feature Extractors (Essentia E23, Omnizart/VGG ALL62) -> Standardized Base Features.
  GP Engine: Population of expression trees -> Mutation/Crossover -> Fitness Evaluation (XGBoost on augmented set) -> Selection + Parsimony Penalty.
  Output: Evolved Composite Features -> Final XGBoost Classifier -> Tag Predictions.
  Auxiliary: DEAP framework, Protected Primitives, NaN Sanitization.
- Critical path:
  1. Extract and normalize base features for target dataset.
  2. Initialize GP population with ramped half-and-half trees.
  3. Loop: Evaluate fitness via XGBoost validation AUC/ACC on Base + Candidate; apply parsimony penalty; select parents; generate offspring.
  4. Terminate on stagnation or max generations; export best expressions.
  5. Retrain XGBoost on Base + Top M Expressions for deployment.
- Design tradeoffs:
  - Search Cost vs. Inference Cost: GP incurs heavy training-time computation (training XGBoost per candidate); inference is cheap (evaluating formulas).
  - Interpretability vs. Max Accuracy: Parsimony and XGBoost constraints may yield slightly lower accuracy than deep learning baselines in exchange for transparent formulas.
  - Operator Set vs. Bloat Risk: Adding complex operators increases expressivity but raises the risk of uninterpretable bloat; requires careful lambda tuning.
- Failure signatures:
  - Bloat: Expression depth grows rapidly while fitness plateaus or declines; check tree size distribution.
  - Stagnation: No improvement over 15 generations; may indicate population diversity loss or local optimum trap.
  - Numerical Instability: Frequent NaN/Inf values in evaluation; suggests insufficient protected primitives or poor feature scaling.
  - Overfitting: Large gap between train and validation fitness; indicates GP is memorizing noise rather than learning generalizable interactions.
- First 3 experiments:
  1. Baseline Replication: Run E23 and ALL62 with XGBoost to reproduce baseline scores and verify data pipelines.
  2. Parsimony Sensitivity Sweep: Run GP with varying lambda values to map the trade-off curve between expression complexity and final AUC/ACC.
  3. Operator Ablation: Disable conditional operators and trigonometric functions to quantify their individual contribution to performance gains and expression diversity.

## Open Questions the Paper Calls Out
### Open Question 1
- Question: Do the GP-evolved composite features generalize across different music tagging datasets or tasks without requiring full re-evolution?
- Basis in paper: [inferred] The authors train GP separately for MTG-Jamendo and GTZAN, noting distinct feature synergies per dataset. Cross-dataset transferability is never evaluated.
- Why unresolved: Symbolic regression is highly sensitive to training distributions; evolved expressions may capture dataset-specific annotation biases or genre distributions rather than universal perceptual rules.
- What evidence would resolve it: Systematically testing top-evolved expressions from one dataset on multiple unseen tagging benchmarks, comparing zero-shot accuracy against re-evolved baselines and measuring structural stability across domains.

### Open Question 2
- Question: Can GP-evolved composite features preserve their interpretability and performance gains when integrated into deep learning architectures?
- Basis in paper: [inferred] The pipeline is evaluated exclusively with XGBoost. The introduction highlights that deep models still outperform the approach on GTZAN but lack transparency, leaving hybrid integration unexplored.
- Why unresolved: Deep networks may nonlinearly recombine or obscure the explicit mathematical relationships discovered by GP, potentially negating the interpretability advantage during end-to-end training.
- What evidence would resolve it: Replacing raw base features with GP-composites as input layers or attention priors in CNN/Transformer models, tracking gradient attribution consistency and final tag accuracy to verify preserved transparency.

### Open Question 3
- Question: How does GP search efficiency and expression quality scale when the base feature set expands to high-dimensional audio representations?
- Basis in paper: [inferred] Experiments use only 23D and 62D feature sets. The combinatorial explosion of GP terminal sets is a known constraint, but scalability to modern high-dimensional descriptors is not tested.
- Why unresolved: The reported rapid convergence may degrade sharply as terminal diversity increases, leading to bloat or intractable search spaces without adaptive parsimony tuning.
- What evidence would resolve it: Profiling runtime, hit-rates, and final fitness across systematically expanded feature sets, benchmarking against adaptive complexity penalties and pruning strategies.

## Limitations
- Marginal performance gains are tightly coupled to XGBoost inductive bias and may not scale to opaque deep baselines.
- Unreported data split procedures and extended feature preprocessing details hinder faithful replication and raise cross-dataset generalizability risks.
- The combinatorial search space remains untested at higher dimensionalities, leaving scalability to modern high-dimensional audio representations uncertain.
